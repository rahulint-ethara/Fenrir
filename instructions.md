# Online Gaming Website - Task Instructions

## Project Summary

Create a responsive online gaming website for a casual browser-game portal. The website must allow visitors to browse sample games, filter games by category, open a demo play screen, and understand the purpose of the portal from a polished landing page.

The deliverable is a frontend MVP. It must focus on layout quality, responsive behavior, clear navigation, game-card presentation, and basic interactivity.

## Primary Objective

Build a browser-ready online gaming website using HTML, CSS, and JavaScript. React is allowed only if the final project can still be run with clear setup instructions.

## Required Pages or Sections

The website must include a homepage hero section.

The website must include a navigation bar.

The website must include a featured games section.

The website must include a game catalog section.

The website must include a category filter interface.

The website must include a game detail or demo play screen.

The website must include an About section.

The website must include a Contact section.

The website must include a footer.

## Game Catalog Requirements

The catalog must show at least six sample games.

Each game card must show a thumbnail.

Each game card must show a game title.

Each game card must show a game category.

Each game card must show a visible play button.

The category filter must include Action, Puzzle, Arcade, Racing, and Casual.

Selecting a category must update the visible game list.

## Demo Play Requirements

The website must include at least one playable JavaScript demo or a clearly interactive game placeholder.

The play area must have visible game instructions.

The play area must include a way to return to the catalog.

The demo must run in the browser without external paid services.

## Visual Direction

Use a modern arcade-inspired visual style.

Use a dark background theme.

Use bright accent colors for buttons, cards, and active states.

Use readable typography across all screen sizes.

Use hover states or simple transitions for interactive elements.

## Responsive Requirements

The layout must work on desktop screens.

The layout must work on tablet screens.

The layout must work on mobile screens.

The navigation must remain usable on mobile screens.

Game cards must reflow cleanly on smaller screens.

## Technical Requirements

Use HTML, CSS, and JavaScript as the primary frontend stack.

Use clean folder organization.

Use local project files for styles and scripts.

Use semantic HTML where practical.

Avoid console errors during normal use.

Avoid broken internal links.

Avoid broken image references.

## Asset Rules

Use only self-created, public-domain, CC0, or CC-BY assets.

Do not use copyrighted game characters.

Do not use trademarked game logos.

Do not use screenshots from commercial games.

List every third-party asset in the credits file if any external assets are used.

## Out of Scope

Real-money gaming is not required.

User login is not required.

Payment processing is not required.

Online multiplayer is not required.

A backend dashboard is not required.

Leaderboard persistence is not required.

Mobile app export is not required.

## Required Deliverables

Submit the complete editable source project.

Submit a README.md file with setup instructions.

Submit all image, icon, audio, or font assets used by the website.

Submit an asset credits file if third-party assets are used.

Submit the project in a public GitHub repository.

## Final Submission Validation

Confirm that the website opens locally.

Confirm that the navigation works.

Confirm that the category filter works.

Confirm that the play screen opens.

Confirm that the layout is responsive.

Confirm that the README.md file is included.

Confirm that no copyrighted game assets are included.
